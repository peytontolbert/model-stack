# Governance Package

`governance/` generates the metadata and audit artifacts that should travel with
model exports, checkpoints, and serving releases:

- model cards
- SPDX SBOMs
- SHA-256 checksum files
- optional Ed25519 signatures
- reproducibility receipts
- lineage graphs

The package is intentionally lightweight and file-oriented. It can be used from
Python APIs or through `python -m governance`.

## File Map

| File | Responsibility |
| --- | --- |
| `card.py` | Render `MODEL_CARD.md` and machine-readable `MODEL_CARD.json`. |
| `sbom.py` | Generate SPDX 2.3 JSON from installed Python packages. |
| `signature.py` | Compute SHA-256 checksums and optional Ed25519 signatures. |
| `receipt.py` | Capture reproducibility metadata: artifacts, system fingerprint, selected environment, git info, packages. |
| `lineage.py` | Write DOT lineage graphs from training metadata. |
| `cli.py` | Command-line entry point. |
| `utils.py` | Shared filesystem, JSON, git, and environment helpers. |

## CLI

```bash
python -m governance card path/to/model.safetensors \
  --metadata metadata.json \
  --out artifacts/MODEL_CARD.md

python -m governance sbom --out artifacts/SBOM.spdx.json --name model-release

python -m governance sign \
  --files artifacts/model.safetensors artifacts/MODEL_CARD.md \
  --out artifacts/CHECKSUMS.sha256

python -m governance receipt \
  --artifacts artifacts/model.safetensors artifacts/SBOM.spdx.json \
  --out artifacts/RECEIPT.json

python -m governance lineage \
  --metadata training-metadata.json \
  --out artifacts/LINEAGE.dot

python -m governance verify artifacts/model.safetensors
```

`verify` checks for the expected governance files next to the artifact:

- `MODEL_CARD.md`
- `MODEL_CARD.json`
- `SBOM.spdx.json`
- `CHECKSUMS.sha256`
- `RECEIPT.json`
- `LINEAGE.dot`

## Metadata Inputs

Model card metadata may include:

- `model.name`
- `model.version`
- `model.base`
- `license`
- `datasets`
- `training`
- `evaluation`
- `limitations`
- `ethical_considerations`
- `intended_use`
- `sources`

Lineage metadata uses `model`, `datasets`, and `model.base` to build a simple
dataset/checkpoint/model graph.

## Safety And Privacy Rules

`receipt.py` records a filtered environment snapshot. It only includes prefixes
that are useful for reproducibility, such as `CUDA`, `CUDNN`, `PYTORCH`, `HF_`,
`TRANSFORMERS_`, `NCCL`, `OMPI`, and `CI`. Do not broaden this list without
checking for secret leakage risk.

Private signing keys are never generated by this package. If `--key` is passed,
the key file must contain either a raw 32-byte Ed25519 private key or a base64
encoding of one.

## Registry Integration

Governance artifacts can be registered with `registry.ArtifactRegistry` after
generation. The registry records checksums and lifecycle stages; governance owns
the audit files themselves.
