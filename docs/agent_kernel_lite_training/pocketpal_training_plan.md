# PocketPal Training Plan

PocketPal should be trained as a local personal-agent controller, not as a model
that memorizes every user artifact. The stable model learns the control
contract; the user's changing information lives in slots, local memory packs,
retrieval context, and installed extension manifests.

## Control Contract

The encoder prompt should compile:

1. user profile slots
2. recent chat history
3. local memory or retrieved user context
4. installed extension manifests and approval policy
5. the current user request

The decoder should emit a structured decision using existing AgentKernel Lite
actions:

- `respond`: answer directly from the prompt/context.
- `gather_context`: retrieve local user context or research evidence.
- `save_memory`: persist a user-approved preference or fact locally.
- `extension_request`: request approval to use an installed extension.
- `ask_user`: ask for a missing configuration slot or ambiguous intent.

## Dataset Mixture

Use `scripts/agent_kernel_lite/build_agentkernel_lite_encdec_dataset.py`.

PocketPal-specific examples are generated by `--max-user-slot-examples` and
cover:

- slot-conditioned responses
- slot updates
- privacy boundaries
- local memory writes
- installed extension routing
- extension-result summaries
- missing-slot questions
- answers grounded in generic user context, not only papers

The generator is deterministic and varies focus area, time horizon, tone,
preference, constraint, privacy rule, context type, missing slot, memory kind,
and installed extension. This is intentional: large values such as
`--max-user-slot-examples 20000` should create a broad controller curriculum
instead of repeating the same few examples.

Research-paper data remains useful, but it should become one source type inside
the wider local context system.

## Stage 2: Mixed PocketPal Model

Build a mixed dataset with app docs, paper context, and user-slot examples:

```bash
/home/peyton/miniconda3/envs/ai/bin/python scripts/agent_kernel_lite/build_agentkernel_lite_encdec_dataset.py \
  --output-dir artifacts/pocketpal_mixed_dataset \
  --docs-root docs \
  --max-doc-files 48 \
  --paper-text-path /path/to/papers.jsonl \
  --max-paper-text-examples 50000 \
  --max-user-slot-examples 20000 \
  --objective chat
```

For a stronger user-focused model, add curated external agentic data before the
final 100M run. See `docs/pocketpal_agentic_datasets.md` for the recommended
mix:

- 40% PocketPal synthetic slots/memory/extensions
- 25% function-calling data converted to extension requests
- 20% multi-turn tool/user traces converted to decisions
- 10% research/paper curriculum
- 5% general agentic instruction SFT

External agentic rows can be converted with
`scripts/convert_agentic_dataset_to_pocketpal.py`, then merged with the
PocketPal/controller dataset before training.

## Agent Quality Gate

Before training a larger mixed run, build a small deterministic agent-quality
set that measures the core PocketPal contract: the active user-configured agent
instruction must control the response, while retrieval/tool/action policies
control routing.

```bash
/home/peyton/miniconda3/envs/ai/bin/python scripts/agent_kernel_lite/build_pocketpal_agent_quality_dataset.py \
  --output-dir artifacts/pocketpal_agent_quality_dataset
```

Use this dataset as both a high-weight training shard and a regression eval.
It intentionally includes arbitrary user-written agent instructions rather than
runtime-hardcoded agent types. Passing this eval matters more than generic
perplexity: if the model ignores the active agent instruction, PocketPal is not
working as a personal on-device agent.

Train the browser-target model:

```bash
/home/peyton/miniconda3/envs/ai/bin/python scripts/agent_kernel_lite/train_agentkernel_lite_encdec.py \
  --dataset-manifest artifacts/pocketpal_mixed_dataset/agentkernel_lite_encdec_dataset_manifest.json \
  --output-dir artifacts/pocketpal_100m_bitnet \
  --preset agentkernel-lite-100m \
  --tokenizer-kind agentkernel-bpe \
  --tokenizer-vocab-size 32768 \
  --tokenizer-max-texts 200000 \
  --max-encoder-tokens 1024 \
  --max-decoder-tokens 512 \
  --batch-size 2 \
  --max-steps 10000 \
  --bitnet-qat 1 \
  --dry-run 0 \
  --export-browser-bitnet 1
```

## Stage 3: User Adaptation

Do not fine-tune the whole model on private user data by default. Adapt users by:

- editing profile slots
- saving local memories
- importing local retrieval packs
- enabling or disabling extensions
- optionally fine-tuning only small adapters or embeddings later

That keeps PocketPal fast, reversible, and local-first.
